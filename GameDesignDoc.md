# 漢字ローグライク ゲームデザイン仕様書

## 概要
2Dトップダウン型の漢字カードバトル・ローグライクゲーム。
プレイヤーは漢字カードを駆使してフィールドを探索し、敵と戦い、カードを合体させながら強くなる。

---

## 基本システム

### HP システム
- **プレイヤー最大HP**: 50（初期値）
- **ダメージ計算**: `実ダメージ = max(0, 攻撃力 - 防御バフ)`
- **オーバーヒール**: 回復時に最大HPを超えることができる。上限は `maxHP × 2`
  - 通常HP範囲（0〜maxHP）: HPバーは緑色
  - オーバーヒール状態（maxHP超過）: HPバーは金色に変化
  - 30%以下: HPバーは赤色に変化

### HPバーアニメーション仕様
- **メインバー Tween**: HP変動時、0.35秒かけて `ease-out quad` で滑らかに変化
- **遅延バー（格ゲー風）**: ダメージ受けた際、メインバーが先に減り、
  0.25秒待機後に0.45秒かけてリニアに追従する赤いバーを背後に表示
- 戦闘開始・リセット時は即時セット（`SetHPImmediate`）

## バトルシステム

### ターン制戦闘フロー
1. プレイヤーターン：手札からカードを使用
2. ターン終了ボタンで敵ターンへ
3. 敵攻撃 → プレイヤーターンへ戻る

### APレスシステム（AP廃止）
- **APシステムは完全廃止**。すべてのカードはコスト0で使い放題。
- `GameManager.EnforceCardBalancePatches()` が全カードコストを0に統一する。
- カードに表示されるコスト表示も非表示に変更済み。

### 超循環化ドローシステム
カードを使うほどドローが加速する自己完結型循環システム：
- **カード使用 → 1枚ドロー**：`PlayCard()` 実行後、手札が `initialHandSize` 未満なら1枚補充
- **合体成功 → 2枚ドロー**：Enemy Fusion Break や手動合体成功時に最大2枚補充
- **Mirror Clash（相殺）→ 1枚ドロー**：相殺成功時にドロー補充
- 目的：カードを積極的に使い続けることで自然にデッキが循環し、攻撃的プレイが報酬される

### ターン終了
- **プレイヤーが「ターン終了」ボタンを手動で押した場合のみ敵のターンへ移行する**
- APが0になっても自動でターンは終了しない（オートターンエンド廃止）
- 意図的にターンを引き延ばしてコンボを狙うプレイが可能

### カードタイプ
| タイプ | 効果 |
|--------|------|
| Attack | 敵にダメージ |
| Defense | 防御バフ付与 |
| Heal | HP回復（オーバーヒール対応） |
| Buff | 攻撃バフ付与 |
| Special | ダメージ + 60%吸血回復 |
| Draw | カード2枚追加ドロー |
| AttackAll | 全体攻撃 |
| Stun | 敵スタン（次ターン行動不能） |
| Shop | 5枚のランダムカードから1枚選んで手札に追加 |

## シールド（命綱）システム

### 概要
デュエルマスターズ風の防御システム。戦闘開始時にデッキから3枚のカードが「シールド」として裏向きに置かれ、HPを守る命綱になる。

### 仕様
- **初期シールド枚数**: 3枚（`GameManager.maxShields = 3`）
- **シールド生成**: `InitializeBattleDeck()` 時にdrawPile先頭から3枚を `shields` リストへ移動
- **シールドトリガー**: プレイヤーがダメージを受けた際、HPへのダメージより先にシールドが発動する
  - シールドが1枚破壊され、そのカードが**手札**（空きあり）または**drawPile先頭**に追加される
  - HPへのダメージは完全に0になる（シールド=完全防御）
  - ログに「シールドトリガー！『カード名』が手札に加わった！」（シアン色）を表示
- **シールド切れ後**: シールドが0枚になったらダメージが通常通りHPへ
- **UI**: 画面左上・HPバー下に青い裏向きカード（「?」）として表示。残枚数分だけ表示され、なくなると灰色の空スロットに変化

## コンボシステム

### 熟語コンボ
直前に使ったカードと合わせて熟語が成立した場合、ボーナスダメージ。
例: 「火」→「炎」で「火炎」コンボ（+15ダメージ）

### 属性チェインコンボ
同属性カードを連続使用するたびに `チェイン数 × 2` のボーナスダメージ。

### 漢字ブレイク：相殺（Mirror Clash）
使用カードの漢字と敵の漢字が同一の場合、ダメージ3倍 + ボーナス1ドロー。

### 漢字ブレイク：構成数マウント（Overpower）
使用カードの画数が敵より多い場合、ダメージ1.5倍。

### 同字連撃コンボ（新規実装）
同一の漢字カードを連続して攻撃に使用した場合、以下の倍率を適用：

| 連撃数 | 倍率 | エフェクト |
|--------|------|-----------|
| 初回（1回目） | ×1.0 | なし |
| 2連続 | ×1.5 |「2 COMBO! ×1.5 DAMAGE!」ポップアップ（ピンク色） |
| 3連続以上 | ×2.0 | 「N COMBO!! ×2.0 DAMAGE!」ポップアップ（紫色）+ 確定クリティカル演出 |

- 対象: Attack / AttackAll / Special カードのみ
- リセット条件: 違う漢字を使用 / ターン終了 / 新ターン開始 / 戦闘開始

## 攻撃ターゲット選択（Attack Prediction Button）
- **全カードタイプ**をタップするとカードがハイライトされ、画面中央（敵とカードスロットの中間）に**予測ボタン**が表示される
- 攻撃系（Attack/AttackAll/Special）→赤いボタン、その他→青いボタン
- ボタンに表示されるテキスト例：
  - Attack/AttackAll: `⚔ N DMG` / `⚔ N DMG (ALL)`
  - Special: `⚔ N DMG / +M HP`
  - Heal: `+N HP`
  - Defense: `+N DEF`
  - Buff: `+N ATK`
  - Stun: `1 STUN`
  - Draw: `N DRAW`
- 予測ダメージは `effectValue + attackModifier + 攻撃バフ` を計算し、Mirror Clash（×3）/ Overpower（×1.5）ボーナスも反映
- **コンボ・相殺が発動する場合はボタンに特別テキストを表示**：
  - 相殺（CLASH）: `3.0x DMG (CLASH)` + 実ダメージ
  - コンボ（COMBO）: `1.5x DMG (COMBO)` または `2.0x DMG (COMBO)` + 実ダメージ
- ボタンをタップすることで効果が実行される
- 別カードタップまたは戦闘終了でボタンは消去される

## 手札バッジUI（コンボ・相殺チャンス表示）
- 手札カードの上部に小さなバッジ（背景色付きテキスト）を表示してプレイヤーに戦況を通知する
- **相殺バッジ（赤）**: 手札カードの漢字が現在の敵の漢字と同一の場合、そのカード上に「相殺」と表示
- **コンボバッジ（紫）**: 直前に使用したカードと同じ漢字が手札にある場合、そのカード上に「コンボ」と表示
- バッジはUpdateHandUI呼び出し時に毎回再生成される（BattleUI.cs: AttachBattleBadge / CreateBadge）

## 敵との強制合体（Enemy Fusion Break）
カードを敵に向かって合体操作すると、敵の漢字と合体できる場合がある。
成功すると敵がスタン + ボーナス2ドロー。

## フィールドシステム
- 2Dトップダウン視点
- プレイヤーキャラクター「人」をWASDで移動
- 敵シンボルに接触で戦闘開始
- 勝利後に敵シンボル消滅、フィールドの探索継続

### 階層システム（Floor System）
- フィールド右上隅に**階段（▽）**が配置される。ラベルは「F{n+1}↓」
- 階段に乗るとフロアが進み、新たなフィールドが生成される（`currentFloor++`）
- 現在フロア数はフィールドHUD「F{n} 鬼:{defeated}/{threshold}」として表示される
- 敵のHP・ATKはフロアに応じてスケーリングされる：`× (1 + (floor - 1) × 0.3)`
  - 例: F3では敵が1.6倍強化

### 鬼の追跡者（Oni Tracker）システム
- **発生条件**: 同フロアで **5体撃破** すると鬼が出現する（OniSpawnThreshold=5）
- 鬼はプレイヤーと逆側の角にスポーン。「鬼」の漢字、暗赤色背景、赤いアウラ演出
- 鬼は **0.8秒ごとに1マス** プレイヤーに向かって移動する（グリッドベース追跡）
- 接触するとボス戦が開始される
  - 鬼のHP: `80 + floor × 20`、ATK: `12 + floor × 3`、タイプ: Boss
- 鬼はフロアを進んでも一定条件でリセットされる

## インベントリ・デッキシステム
- インベントリ最大30枚
- バトル用デッキ（循環システム）：デッキを使い切ったら捨て札をシャッフルして山札に補充
- 初期手札：5枚ドロー（差分補充方式）

### 手札持ち越しシステム（Hand Carryover）
- 戦闘に**勝利した際、手札はそのまま次の戦闘に持ち越される**（手札クリアなし）
- 次戦闘開始時の `InitializeBattleDeck()` は、すでに手札にあるカードを除いてデッキを再構築する
- 手札が `initialHandSize` に満たない分だけ山札から差分補充される
- 敗北（ゲームオーバー）時はシーンリロードのため手札は消去される

---

## ショップ・合体（Fusion）
- 合体コスト: 20G
- 2枚または3枚のカードを合体して新カードを生成
- 合体レシピはKanjiFusionDatabase.asset で管理

### バトル合体演出（Grand Fusion Sequence）
合体ボタン押下時に画面中央でド派手な演出が0.75秒以内で展開される：
1. 暗転オーバーレイが出現し操作をブロック
2. 合体元2枚のカードが左右から画面中央に向かってEaseInスライド（0.28秒）
3. 衝突瞬間にCFXR爆発エフェクト＋カメラシェイク＋全画面フラッシュ
4. 結果カードがスケール0→バウンスアニメーションで出現（0.4秒）
5. 0.38秒表示後にフェードアウト → 結果カードが手札に追加
- 演出中はフューズボタンが無効化され重複操作を防止
- 合体後は2枚ボーナスドロー

## チュートリアルシステム（TitleScreenManager）

### 起動フロー
- **初回起動時のみ**（`PlayerPrefs("TutorialCompleted") == 0`）タイトル画面を表示する
- 二回目以降はタイトルをスキップして直接フィールドへ

### タイトル画面
- 「漢字ローグライク」（黄色・太字・大文字）+ サブタイトル + 点滅「CLICK TO START」
- どのキーまたはマウスクリックで開始

### チュートリアル戦闘
1. タイトル画面フェードアウト
2. HelpPanelを表示（ゲームルール説明）、タイムスケール0で一時停止
3. 「理解した！戦闘開始！」ボタンでHelpPanelを閉じてタイムスケール1に戻す
4. チュートリアル専用弱い敵（HP:12, ATK:2, 「弱」）と戦闘開始
5. 勝利後: `PlayerPrefs("TutorialCompleted") = 1` に設定してフィールドへ移行
   - 以降は起動してもタイトル画面は表示されない

## BGM システム（AudioManager）
- `AudioManager` シングルトンが GameManager オブジェクトに付属し、BGMを一元管理する
- **通常バトルBGM**: `404FreezeCode.mp3` と `memento_loop.ogg` からランダムで選曲
  - 各曲のBPMに応じてBPM同期変数を自動切り替え（404FreezeCode: 152BPM, memento_loop: 120BPM）
- **狼ボスBGM（フェーズ別）**:
  - **第1形態**: `404FreezeCode.mp3` を固定再生（ランダム選曲しない）
  - **第2形態移行時**: `Loneryboy.mp3` にフェード切替
- **フィールドBGM**: フィールド復帰時（`ReturnToField()`）に切り替え。fieldBGMが未設定の場合はBGM停止
- 音量は `bgmVolume`（初期: 0.7）でコントロール
- BGMクリップがインスペクタ未設定の場合、`Resources.FindObjectsOfTypeAll`で自動探索

---

## 狼ボスシステム（WolfBossManager）
- **第一形態**: 初期HPはプレイヤーの現在HPと同値に設定
- **第二形態への移行**: 第一形態HP0時、HPを99に設定し「君となら本気で戦えそうだ」ダイアログを表示。BGMが `Loneryboy` に切替
- **戦闘中回復なし**: 狼ボスはHPを回復しない
- **特殊カード「血」**: 3ターン毎にプレイヤーの手札のランダム1枚が「血」カードに変化
  - 血カード効果: 敵に3ダメージ + 自分にも3ダメージ（自傷）
- **手札破壊**: 一定確率でプレイヤーの手札を「404 ERROR」（使用不可）に変化

---

## ステージ進行システム
- 各階層の**階段（次フロアへのマス）は初期状態でロック**されている
- **ステージボスを撃破**すると階段が解放され、画面に「次の階層への道が開かれた！」通知が表示される
- ボスを倒さない限り、次の階層には進めない
- **フロア3以降のボスは必ず「狼（WolfBoss）」に固定**される（F1〜2は通常ボス）
  - 狼ボスは動的生成（HP=150+, ATK=18+）、`isWolfBoss = true` フラグにより専用システム発動

---

## 鬼の追跡者（Oni Tracker）システム
- **発生条件**: 同フロアで **10回戦闘** を行うと鬼が出現する（OniSpawnThreshold=10）
- フィールドUIに「**鬼出現まであと X 戦**」カウントダウンテキストを表示
  - 残り5戦以下でテキストが赤く変化（警告演出）
  - 階層を移動するとカウントはリセット
- 鬼はプレイヤーと逆側の角にスポーン。「鬼」の漢字、暗赤色背景、赤いアウラ演出
- 鬼は **0.8秒ごとに1マス** プレイヤーに向かって移動する（グリッドベース追跡）
- 接触するとボス戦が開始される
  - 鬼のHP: `80 + floor × 20`、ATK: `12 + floor × 3`、タイプ: Boss
  - **`isOniBoss = true`** により専用BGM（KARMA1Loop）・カウントダウンシステムが発動
  - 毎ターン「九→八→七…→零」とカウントダウン。**零になるとプレイヤーに即死ダメージ**
  - 鬼のターンは通常攻撃なし（カウントダウンのみ）

---

## 逃走システム（Flee Command）
- バトル画面に「**逃げる**」ボタンを表示（ターン終了ボタンの下、ダークレッド）
- 逃走時は**カード1枚がランダムに永久削除（ロスト）**される
- 逃走成功率は100%（確定逃走）
- **ボス戦からの逃走**: 可能だが、フィールドのボスシンボルは消滅せず残る。ボスを倒さない限り次の階層へは行けない

---

## 戦闘勝利・リザルト画面
- 敵を倒した直後、フィールドに戻る前に**ペルソナ風リザルトパネル**を約1.5秒表示する
- 表示内容: 「勝　利！」（金色大文字） + 「獲得: 『漢字』」（橙色）
- **ドロップシステム**:
  - `EnemyData.dropCard` が設定されている場合はそのカードをドロップ
  - 未設定の雑魚敵（`EnemyType.Normal`）は**敵の漢字と同名の攻撃カードを自動ドロップ**（攻撃力3、AP1）
  - インベントリが満杯の場合はドロップ失敗（ログに表示）
- **捨てるボタン**:
  - バトル画面の右側に「捨てる（AP不要）」ボタンを配置
  - カードをクリックで選択後、このボタンを押すとAPなしで手札から削除（捨て札へ）
  - 次のターンのドローフェーズで補充される

## UI 可読性
- 全バトルUIテキスト（HP・AP・敵名・戦闘ログ）に `UnityEngine.UI.Outline` コンポーネントを付与
- 攻撃ボタン、敵名テキストなどキャンバス上の重要要素に黒アウトラインを適用することで背景に依存せず可読性を確保

## ゲームオーバー・リトライ
- プレイヤーHP=0でゲームオーバー
- 「最初から」ボタンでシーンリロード
- リロード時にすべての状態を初期化（シングルトン・静的変数含む）
