# 漢字ローグライク ゲームデザイン仕様書

## 概要
2Dトップダウン型の漢字カードバトル・ローグライクゲーム。
プレイヤーは漢字カードを駆使してフィールドを探索し、敵と戦い、カードを合体させながら強くなる。

---

## 基本システム

### ライフシステム（シールド一本化）
- **プレイヤーのHPシステムは廃止**。ライフはすべて「シールドの枚数」で管理する。
- **ダメージ計算**: `実ダメージ = max(0, 攻撃力 - 防御バフ)`
  - ダメージを受ける → シールドが1枚減る（減ったカードが手札またはdrawPileに入る）
  - シールドが0枚の状態でダメージを受けると即座にゲームオーバー
- **回復**: 回復カード使用 / 特殊カード吸血 / リジェネ状態 → シールドが1枚追加（`maxShields`上限あり）
- **プレイヤーHPバー**: 廃止（敵のHPバーは維持）
- **ゲームオーバー条件**: シールド0枚でダメージを受けた瞬間

---

## バトルシステム

### ターン制戦闘フロー
1. プレイヤーターン：手札からカードを使用
2. ターン終了ボタンで敵ターンへ
3. 敵攻撃 → プレイヤーターンへ戻る

### APシステム（行動値）
- **初期AP / ターン最大AP**: 3（`GameManager.playerStartMana = 3`）
- **ターン開始時**: APが3に全回復（`StartPlayerTurn()` → `playerMana = playerMaxMana`）
- **AP消費コスト**:
  - **AP:1** → Attack / AttackAll / Heal / Special / Stun カード
  - **AP:0** → Buff / Defense / Draw / Shop カード
- **ドローボタン**: AP1消費で山札から1枚引く
- **AP不足時**: AP表示が赤く点滅＋エラーSE再生（`ShowAPError()`）
- **カードコスト表示**: AP:1のカードはコスト枠に「AP:1」と表示
- `GameManager.GetCardAPCost(card)` でカードタイプ別コストを判定

### ドロー・ボーナスシステム
- **カード使用 → 自動ドローなし**：`PlayCard()` 後は自動補充しない。手動ドロー（AP消費）が基本
- **合体成功 → AP+1（1 MORE!）**：手動合体・Enemy Fusion Break 成功時にAP1回復（「1 MORE!」VFX表示）
- **Mirror Clash（相殺）→ 1枚ドロー**：相殺成功時にボーナスドロー1枚（APコストなし）
- 捨てるボタン廃止：手札管理は合体・ドローボタンで行う

### ターン終了
- **プレイヤーが「ターン終了」ボタンを手動で押した場合のみ敵のターンへ移行する**
- APが0になっても自動でターンは終了しない（オートターンエンド廃止）
- 意図的にターンを引き延ばしてコンボを狙うプレイが可能

### カードタイプ
| タイプ | 効果 |
|--------|------|
| Attack | 敵にダメージ |
| Defense | 防御バフ付与 |
| Heal | **シールド追加**（drawPile/discardPileから1枚をshieldsに移動）|
| Buff | 攻撃バフ付与 |
| Special | ダメージ + 60%吸血回復 |
| Draw | カード2枚追加ドロー |
| AttackAll | 全体攻撃 |
| Stun | 敵スタン（次ターン行動不能） |
| Shop | 5枚のランダムカードから1枚選んで手札に追加 |

## シールド（命綱）システム

### 概要
デュエルマスターズ風の防御システム。戦闘開始時にデッキから3枚のカードが「シールド」として裏向きに置かれ、HPを守る命綱になる。

### 仕様
- **初期シールド枚数**: 7枚（`GameManager.maxShields = 7`）
- **シールド生成**: `InitializeBattleDeck()` 時にdrawPile先頭から7枚を `shields` リストへ移動
- **シールドトリガー**: プレイヤーがダメージを受けた際、シールドが発動する
  - シールドが1枚破壊され、そのカードが**手札**（空きあり）または**drawPile先頭**に追加される
  - ダメージは完全に0になる（シールド=完全防御）
  - ログに「シールドトリガー！『カード名』が手札に加わった！」（シアン色）を表示
- **シールド切れ後**: シールドが0枚の状態でダメージを受けると**即ゲームオーバー**
- **回復カード**: 山札または捨て札からカードを1枚取得してシールドに追加（上限まで）
- **UI仕様（小型化）**:
  - 画面左上に青い小型アイコン（38×50px）として最大**10個**まで表示
  - 実際のシールド枚数が10枚を超えた場合、11枚目以降はアイコンを描画せず、右隣に「**+N**」テキストを表示
  - シールドが0枚の場合、「**盾 0枚**」と赤字で表示

## コンボシステム

### 熟語コンボ
直前に使ったカードと合わせて熟語が成立した場合、ボーナスダメージ。
例: 「火」→「炎」で「火炎」コンボ（+15ダメージ）

### 属性チェインコンボ
同属性カードを連続使用するたびに `チェイン数 × 2` のボーナスダメージ。

### 漢字ブレイク：相殺（Mirror Clash）
使用カードの漢字と敵の漢字が同一の場合、ダメージ3倍 + ボーナス1ドロー。

### 漢字ブレイク：構成数マウント（Overpower）
使用カードの画数が敵より多い場合、ダメージ1.5倍。

### 同字連撃コンボ（新規実装）
同一の漢字カードを連続して攻撃に使用した場合、以下の倍率を適用：

| 連撃数 | 倍率 | エフェクト |
|--------|------|-----------|
| 初回（1回目） | ×1.0 | なし |
| 2連続 | ×1.5 |「2 COMBO! ×1.5 DAMAGE!」ポップアップ（ピンク色） |
| 3連続以上 | ×2.0 | 「N COMBO!! ×2.0 DAMAGE!」ポップアップ（紫色）+ 確定クリティカル演出 |

- 対象: Attack / AttackAll / Special カードのみ
- リセット条件: 違う漢字を使用 / ターン終了 / 新ターン開始 / 戦闘開始

## 攻撃ターゲット選択（Attack Prediction Button）
- **全カードタイプ**をタップするとカードがハイライトされ、画面中央（敵とカードスロットの中間）に**予測ボタン**が表示される
- 攻撃系（Attack/AttackAll/Special）→赤いボタン、その他→青いボタン
- ボタンに表示されるテキスト例：
  - Attack/AttackAll: `⚔ N DMG` / `⚔ N DMG (ALL)`
  - Special: `⚔ N DMG / +M HP`
  - Heal: `+N HP`
  - Defense: `+N DEF`
  - Buff: `+N ATK`
  - Stun: `1 STUN`
  - Draw: `N DRAW`
- 予測ダメージは `effectValue + attackModifier + 攻撃バフ` を計算し、Mirror Clash（×3）/ Overpower（×1.5）ボーナスも反映
- **コンボ・相殺が発動する場合はボタンに特別テキストを表示**：
  - 相殺（CLASH）: `3.0x DMG (CLASH)` + 実ダメージ
  - コンボ（COMBO）: `1.5x DMG (COMBO)` または `2.0x DMG (COMBO)` + 実ダメージ
- ボタンをタップすることで効果が実行される
- 別カードタップまたは戦闘終了でボタンは消去される

## 手札バッジUI（コンボ・相殺チャンス表示）
- 手札カードの上部に小さなバッジ（背景色付きテキスト）を表示してプレイヤーに戦況を通知する
- **相殺バッジ（赤）**: 手札カードの漢字が現在の敵の漢字と同一の場合、そのカード上に「相殺」と表示
- **コンボバッジ（紫）**: 直前に使用したカードと同じ漢字が手札にある場合、そのカード上に「コンボ」と表示
- バッジはUpdateHandUI呼び出し時に毎回再生成される（BattleUI.cs: AttachBattleBadge / CreateBadge）

## 敵との強制合体（Enemy Fusion Break）
カードを敵に向かって合体操作すると、敵の漢字と合体できる場合がある。
成功すると敵がスタン + ボーナス2ドロー。

## ルートマップシステム（Slay the Spire式）

### 概要
2Dトップダウンフィールドに代わる、縦スクロール型ルート選択マップ。`RouteMapManager.cs` が管理。

### 構成
- **全7ステージ**（ステージ0〜5: 通常、ステージ6: ボス）
- 各ステージに**3つのノード**（ただしボスは1ノード）
- プレイヤーは現在ステージのノードから1つタップして選択

### UI仕様
- タイトル「▲ ルート選択 ▲」: 画面上部固定（fontSize 30・太字）
- 「◀ 選択してください ▶」: タイトル直下の固定エリアに配置（fontSize 21）。ノードと重ならない
- ノードテキスト（アイコン＋ラベル）: fontSize 18（スマホでの視認性確保）

### ノードタイプ
| タイプ | 説明 |
|--------|------|
| Battle（⚔） | 通常戦闘 |
| EliteBattle（⚔+） | 強化敵との戦闘（HP35, ATK2の「勇」） |
| Shop（店） | コイン20Gまたはシールド1枚でカード1枚購入 |
| OniArea（鬼） | 鬼との高難易度ボス戦（HP80, ATK2, `isOniBoss=true`） |
| TimedBattle（⏱） | 30秒カウントダウン付き戦闘。時間切れで強制敗北 |
| Lottery（★） | 運試し（シールド×2 / ランダムカード / カード複製 / 悪玉+1 / コイン+30G） |
| ShieldAdd（盾） | シールド1枚追加 |
| CardDuplicate（✦） | 手持ちカードから1枚複製して追加 |
| Boss（狼） | ステージ6固定ボス：狼（HP150, ATK2, `isWolfBoss=true`） |

### 悪玉（Akudama）システム
- `RouteMapManager.AkudamaCount`（static）で管理
- 福引きでハズレを引くと悪玉+1
- 悪玉2個消費して福引きリロールが可能
- 将来的にバフ・デバフ効果に活用予定

### 進行ルール
- 選んだノードを訪問すると `isVisited=true`、次ステージの接続ノードが`isUnlocked=true`になる
- 戦闘系ノードは `BattleManager.ReturnToField()` → `RouteMapManager.OnBattleWon()` で currentStage が進む
- 非戦闘系ノード（Shop / ShieldAdd / Lottery / CardDuplicate）は内部で `ReturnToMap()` を呼ぶ

## フィールドシステム（旧）
※ルートマップ導入後はデフォルトで `RouteMapManager` が使用される。`GameManager.routeMapManager` が null の場合のみ `FieldManager` が使われる。

### 階層システム（Floor System）
- フィールド右上隅に**階段（▽）**が配置される。ラベルは「F{n+1}↓」
- 階段に乗るとフロアが進み、新たなフィールドが生成される（`currentFloor++`）
- 現在フロア数はフィールドHUD「F{n} 鬼:{defeated}/{threshold}」として表示される
- 敵のHP・ATKはフロアに応じてスケーリングされる：`× (1 + (floor - 1) × 0.3)`
  - 例: F3では敵が1.6倍強化

### 鬼の追跡者（Oni Tracker）システム
- **発生条件**: 同フロアで **5体撃破** すると鬼が出現する（OniSpawnThreshold=5）
- 鬼はプレイヤーと逆側の角にスポーン。「鬼」の漢字、暗赤色背景、赤いアウラ演出
- 鬼は **0.8秒ごとに1マス** プレイヤーに向かって移動する（グリッドベース追跡）
- 接触するとボス戦が開始される
  - 鬼のHP: `80 + floor × 20`、ATK: `12 + floor × 3`、タイプ: Boss
- 鬼はフロアを進んでも一定条件でリセットされる

---

## インベントリ・デッキシステム
- インベントリ最大30枚
- バトル用デッキ（循環システム）：デッキを使い切ったら捨て札をシャッフルして山札に補充
- 初期手札：5枚ドロー（差分補充方式）

### 手札持ち越しシステム（Hand Carryover）
- 戦闘に**勝利した際、手札はそのまま次の戦闘に持ち越される**（手札クリアなし）
- 次戦闘開始時の `InitializeBattleDeck()` は、すでに手札にあるカードを除いてデッキを再構築する
- 手札が `initialHandSize` に満たない分だけ山札から差分補充される
- 敗北（ゲームオーバー）時はシーンリロードのため手札は消去される

---

## ショップ・合体（Fusion）
- 合体コスト: 20G
- 2枚または3枚のカードを合体して新カードを生成
- 合体レシピはKanjiFusionDatabase.asset で管理

### バトル合体演出（Grand Fusion Sequence）
合体ボタン押下時に画面中央でド派手な演出が0.75秒以内で展開される：
1. 暗転オーバーレイが出現し操作をブロック
2. 合体元2枚のカードが左右から画面中央に向かってEaseInスライド（0.28秒）
3. 衝突瞬間にCFXR爆発エフェクト＋カメラシェイク＋全画面フラッシュ
4. 結果カードがスケール0→バウンスアニメーションで出現（0.4秒）
5. 0.38秒表示後にフェードアウト → 結果カードが手札に追加
- 演出中はフューズボタンが無効化され重複操作を防止
- 合体後は2枚ボーナスドロー

## チュートリアルシステム（TitleScreenManager）

### 起動フロー
- **初回起動時のみ**（`PlayerPrefs("TutorialCompleted") == 0`）タイトル画面を表示する
- 二回目以降はタイトルをスキップして直接フィールドへ

### タイトル画面
- 「漢字ローグライク」（黄色・太字・大文字）+ サブタイトル + 点滅「CLICK TO START」
- どのキーまたはマウスクリックで開始

### チュートリアル戦闘
1. タイトル画面フェードアウト
2. HelpPanelを表示（ゲームルール説明）、タイムスケール0で一時停止
3. 「理解した！戦闘開始！」ボタンでHelpPanelを閉じてタイムスケール1に戻す
4. チュートリアル専用弱い敵（HP:12, ATK:2, 「弱」）と戦闘開始
5. 勝利後: `PlayerPrefs("TutorialCompleted") = 1` に設定してフィールドへ移行
   - 以降は起動してもタイトル画面は表示されない

## BGM システム（AudioManager）
- `AudioManager` シングルトンが GameManager オブジェクトに付属し、BGMを一元管理する
- **通常バトルBGM**: `404FreezeCode.mp3` と `memento_loop.ogg` からランダムで選曲
  - 各曲のBPMに応じてBPM同期変数を自動切り替え（404FreezeCode: 152BPM, memento_loop: 120BPM）
- **狼ボスBGM（フェーズ別）**:
  - **第1形態**: `404FreezeCode.mp3` を固定再生（ランダム選曲しない）
  - **第2形態移行時**: `Loneryboy.mp3` にフェード切替
- **フィールドBGM**: フィールド復帰時（`ReturnToField()`）に切り替え。fieldBGMが未設定の場合はBGM停止
- 音量は `bgmVolume`（初期: 0.7）でコントロール
- BGMクリップがインスペクタ未設定の場合、`Resources.FindObjectsOfTypeAll`で自動探索

---

## 狼ボスシステム（WolfBossManager）
- **第一形態**: 初期HPはプレイヤーの現在HPと同値に設定
- **第二形態への移行**: 第一形態HP0時、HPを99に設定し「君となら本気で戦えそうだ」ダイアログを表示。BGMが `Loneryboy` に切替
- **戦闘中回復なし**: 狼ボスはHPを回復しない
- **特殊カード「血」**: 3ターン毎にプレイヤーの手札のランダム1枚が「血」カードに変化
  - 血カード効果: 敵に3ダメージ + 自分にも3ダメージ（自傷）
- **手札破壊**: 一定確率でプレイヤーの手札を「404 ERROR」（使用不可）に変化

---

## ステージ進行システム
- 各階層の**階段（次フロアへのマス）は初期状態でロック**されている
- **ステージボスを撃破**すると階段が解放され、画面に「次の階層への道が開かれた！」通知が表示される
- ボスを倒さない限り、次の階層には進めない
- **フロア3以降のボスは必ず「狼（WolfBoss）」に固定**される（F1〜2は通常ボス）
  - 狼ボスは動的生成（HP=150+, ATK=18+）、`isWolfBoss = true` フラグにより専用システム発動

---

## 鬼の追跡者（Oni Tracker）システム
- **発生条件**: 同フロアで **10回戦闘** を行うと鬼が出現する（OniSpawnThreshold=10）
- フィールドUIに「**鬼出現まであと X 戦**」カウントダウンテキストを表示
  - 残り5戦以下でテキストが赤く変化（警告演出）
  - 階層を移動するとカウントはリセット
- 鬼はプレイヤーと逆側の角にスポーン。「鬼」の漢字、暗赤色背景、赤いアウラ演出
- 鬼は **0.8秒ごとに1マス** プレイヤーに向かって移動する（グリッドベース追跡）
- 接触するとボス戦が開始される
  - 鬼のHP: `80 + floor × 20`、ATK: `12 + floor × 3`、タイプ: Boss
  - **`isOniBoss = true`** により専用BGM（KARMA1Loop）・カウントダウンシステムが発動
  - 毎ターン「九→八→七…→零」とカウントダウン。**零になるとプレイヤーに即死ダメージ**
  - 鬼のターンは通常攻撃なし（カウントダウンのみ）

---

## 逃走システム（Flee Command）
- バトル画面に「**逃げる**」ボタンを表示（ターン終了ボタンの下、ダークレッド）
- 逃走時は**カード1枚がランダムに永久削除（ロスト）**される
- 逃走成功率は100%（確定逃走）
- **ボス戦からの逃走**: 可能だが、フィールドのボスシンボルは消滅せず残る。ボスを倒さない限り次の階層へは行けない

---

## 戦闘勝利・リザルト画面
- 敵を倒した直後、フィールドに戻る前に**ペルソナ風リザルトパネル**を約1.5秒表示する
- 表示内容: 「勝　利！」（金色大文字） + 「獲得: 『漢字』」（橙色）
- **ドロップシステム**:
  - `EnemyData.dropCard` が設定されている場合はそのカードをドロップ
  - 未設定の雑魚敵（`EnemyType.Normal`）は**敵の漢字と同名の攻撃カードを自動ドロップ**（攻撃力3、AP1）
  - インベントリが満杯の場合はドロップ失敗（ログに表示）
- **捨てるボタン**: 廃止（Slay the Spire式ドロー導入に伴い削除）

## UI 可読性
- 全バトルUIテキスト（HP・AP・敵名・戦闘ログ）に `UnityEngine.UI.Outline` コンポーネントを付与
- 攻撃ボタン、敵名テキストなどキャンバス上の重要要素に黒アウトラインを適用することで背景に依存せず可読性を確保

## UIの親切設計（使用可能時の明滅・AP不足時の視覚フィードバック）

### ハイライト（明滅）演出 ― グローバル時間同期
- **発動条件**: プレイヤーターン中（`BattleState.PlayerTurn`）かつ行動可能な状態の時
- **同期方式**: `Mathf.Sin(Time.unscaledTime * 2.4f)` を使用。すべてのカードとドローボタンが**全く同一の式・同一の位相**で輝度計算するため、画面上の「光るべき対象」がすべて完全同期でフワッと光る
- **ドローボタン**: AP ≥ 1 の時、`BattleUI.Update()` 内でグロウオーバーレイのalpha (0〜0.28) を毎フレーム計算
- **手札カード**: APコストが現在APを超えない場合（使用可能カード）、`CardController.Update()` 内でグロウオーバーレイのalpha (0〜0.32) を毎フレーム計算
- **AP:0のカード（Defense/Draw等）**: 常に使用可能なため、APに関係なく明滅対象
- **実装**: コルーチン廃止。`_drawPulseActive`（BattleUI）および `_glowActive`（CardController）フラグで有効・無効を切り替え

### AP不足時の視覚フィードバック
- **発動条件**: `playerMana < 1`（ドローボタン）または `playerMana < cardAPCost`（攻撃/行動予測ボタン）
- **ドローボタン**: AP=0時に以下を表示
  - 赤い取り消し横線（ボタン中央に水平Image）
  - 「AP不足」赤字テキスト（フォント: AppFont SDF）
  - ボタンを半透明（alpha 0.5）に変更
  - 明滅演出を停止
- **行動予測ボタン（予測DMGボタン）**: AP不足カードをタップ時に以下を表示
  - ボタンを半透明（alpha 0.5）に変更
  - 赤い取り消し横線
  - 「AP不足」赤字テキスト
- **実装**: `BattleUI.UpdateDrawButtonAPState()` が `UpdateStatusUI()` 呼び出し時に毎回状態を同期

### ルート選択画面のデッキ編成ボタン
- **位置**: RouteMapPanel 右上（anchorMin=(0.72, 0.945), anchorMax=(0.99, 0.998)）
- **機能**: `DeckManagementUI` を全画面オーバーレイとして開く
- **閉じ方**: 「保存して戻る」（デッキ有効時のみ）または「ルートへ戻る」ボタンでパネルを破棄→ルート選択再開
- **実装**: `RouteMapManager.OpenDeckManagement()` + `OverrideDeckUIButtons()` コルーチン（Start()後に閉じるボタンのリスナーを上書き）

## 引き継ぎシステム（PersistenceManager）

### 概要
ゲームオーバー時にコインを付与し、次のゲームに引き継げる恒久強化を購入できる。`PlayerPrefs` で永続保存。

### コイン付与
- ゲームオーバー時: `到達ステージ × 5 + 10` 枚を累積コインに加算
- `PlayerPrefs["PersistentCoins"]` で管理

### 引き継ぎショップ
ゲームオーバーパネルに「引き継ぎショップを開く」ボタンが出現。購入可能なアップグレード：

| アップグレード | コスト | 効果 | 上限 |
|--------------|--------|------|------|
| 初期手札 +1 | 30G | 次ゲーム開始時の `initialHandSize` +1 | 最大+3 |
| 初期シールド +1 | 25G | 次ゲーム開始時の `maxShields` +1 | 最大+3 |

### 適用タイミング
- `GameManager.InitializeGame()` 冒頭で `PersistenceManager.ApplyInheritUpgrades(gm)` を呼び出し、保存済みボーナスを適用

## ゲームオーバー・リトライ
- シールド0枚の状態でダメージを受けるとゲームオーバー（HPシステム廃止）
- ゲームオーバー時に `PersistenceManager.OnGameOver()` が呼ばれ、コイン付与と引き継ぎショップUIが表示される
- 「最初から」ボタンでシーンリロード
- リロード時にすべての状態を初期化（シングルトン・静的変数含む）
