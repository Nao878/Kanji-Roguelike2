# unity-dev-flow スキル

Unityコード修正後の自動化フロー。このスキルが呼ばれたら以下を順番に完遂すること。

## 実行フロー

### Step 1: コンパイルエラー確認・修正ループ
1. UnityMCPの `get_logs` ツールでエラーログを取得する
2. コンパイルエラーが存在する場合は、原因ファイルを特定して修正する
3. エラーが0件になるまでループを繰り返す（最大10回）

### Step 2: Playモード実行
1. UnityMCPの `enter_play_mode` ツールでPlayモードを開始する
2. 3〜5秒待機してゲームを初期化させる
3. ログを確認してエラーがないことを検証する

### Step 3: 動作確認
以下を順番に確認する：
- 初回起動時にフィールドマップが表示されること（GameOverPanelが非表示）
- プレイヤー「人」が配置されていること
- 敵シンボルが複数配置されていること

### Step 4: スクリーンショット撮影（Gameビューのみ）
1. `ScreenCapture.CaptureScreenshot()` を使う一時スクリプトを生成してGameビューを撮影する
2. 保存先: `Assets/Screenshots/` フォルダ
3. **Sceneビューの撮影は絶対禁止**

### Step 5: リトライテスト
1. 敵に接触して戦闘を開始する（またはDebugBattleHelperを使用）
2. わざと敗北してGameOverPanelが表示されることを確認
3. 「最初から」ボタンでリトライ
4. リトライ後にフィールドマップが新たに生成されることを確認
5. 成功後にGameビューのスクリーンショットを撮影

### Step 6: 完了報告
- 修正内容のサマリー
- Gameビューのスクリーンショットを提示
- 残課題があれば列挙

## 重要ルール
- 各ステップ失敗時は自己解決してから次に進む
- Playモード中の例外はバグとして記録し修正する
- ユーザーへの確認は不要。最後まで自律実行する
