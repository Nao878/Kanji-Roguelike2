# 漢字ローグライク - Claude Code 開発ガイドライン

## 1. 仕様の単一情報源 (Single Source of Truth)
- ゲームのシステム、カードの効果、合体レシピ、メカニクスについては、必ずルートディレクトリにある `GameDesignDoc.md` を参照すること。
- 推測で仕様を決定しないこと。仕様に変更・追加を加えた場合は、必ず `GameDesignDoc.md` を更新して同期を保つこと。

## 2. 完全自律実行の原則
- **ユーザーへの確認禁止**: 指示されたタスクは、ユーザーに許可を求めず、ファイル検索・コード修正・コンパイル確認まで一気に完遂すること。
- **推測での修正禁止**: バグ修正時は必ず関連するスクリプトファイルを読み込んでから修正すること。
- **エラーの自己解決**: コード変更後は必ずUnityのコンパイルエラーを確認し、エラーが0になるまで自己修正ループを回すこと。
- **Playモードでの確認**: 実装後はツールを用いてPlayモードを実行し、動作確認を行うこと。

## 3. スクリーンショット撮影の厳格なルール
- 実装完了の報告としてスクリーンショットを撮影する際、開発用画面である「Sceneビュー」は絶対に撮影しないこと。
- 必ずプレイヤー視点である「Gameビュー」を撮影すること。外部ツールでGameビューの指定が難しい場合は、Unity標準の `ScreenCapture.CaptureScreenshot()` を用いた一時スクリプトを生成してでもGameビューの状況を提示すること。

## 4. プロジェクト特有の技術ルール
- **VFXの描画順序**: UI(Canvas)と3Dパーティクル(Cartoon FX)が混在するため、Canvasは `Screen Space - Camera` 設定を維持し、Z座標(深度)の不整合によるUI消失バグに常に注意を払うこと。
- **初期化の徹底**: シーンリロード（リトライ時など）に伴うシングルトンや静的変数の残留バグを防ぐため、`Awake` や `Start` でのデータ初期化を徹底すること。
- **BattleManager初期化**: `BattleManager.Start()` で `battleState = Idle`、`gameOverPanel.SetActive(false)` を必ず実行すること。
- **ResetGame()の原則**: シーンリロード前にInstanceをnullにしない。UnityのDestroyオブジェクトはnull等価になるため、`Instance = null` の明示的代入は不要。
